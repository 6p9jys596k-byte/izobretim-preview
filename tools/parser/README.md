# Сборщик каталога

Наполняет магазин товарами двумя способами.

## 1. Прайс партнёра (основной)
Партнёр присылает выгрузку в формате **YML** (стандарт для российских магазинов).
```
node tools/parser/parse.mjs prices tools/parser/samples/example.yml wowhow
```
- читает все `<offer>` из файла;
- берёт название, цену, старую цену, ссылку;
- угадывает категорию по названию (`guessCat`);
- ставит статус `check` — карточку надо проверить.

Пример для проверки уже лежит в `samples/example.yml`.

## 2. Парсер сайта (запасной)
Если прайса нет — черновой сбор прямо с сайта:
```
node tools/parser/parse.mjs crawl https://site.ru/catalog
```
⚠ У каждого сайта своя вёрстка. Селекторы в функции `crawl()` — грубый пример, под конкретного партнёра их правят точечно (открыть страницу → посмотреть, в каких тегах название и цена → подставить).

Важно: тексты и фото с чужого сайта **нельзя копировать как есть** — это черновик. Финальные карточки переписываем под свой стиль.

## Куда кладётся результат
- `assets/js/data-parsed.js` — черновой каталог в формате сайта (`window.STORE_PARSED`);
- `tools/parser/out/catalog.json` — то же в JSON для просмотра.

Дальше проверенные позиции переносим в основной `assets/js/data.js`.

## Добавить нового партнёра
В `parse.mjs` в объект `BRANDS` добавить строку `id: { name, site }`.
При желании — уточнить правила `guessCat` под их ассортимент.
