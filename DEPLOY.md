# Как залить сайт на прод

Коротко: **прод — это ветка `master`**. Как только изменения попадают в `master`, GitHub Pages сам пересобирает сайт за 1–2 минуты. Ничего вручную нажимать не нужно.

- **Адрес сайта (прод):** https://6p9jys596k-byte.github.io/izobretim-preview/
- **Репозиторий:** https://github.com/6p9jys596k-byte/izobretim-preview
- **Сайт под паролем** — вход через ту же форму, что и обычно.

---

## Способ 1. Просто попросить Claude (рекомендую)

В большинстве случаев ничего не надо делать руками. После правок скажите:

> залей на прод

Claude сам сохранит изменения и отправит их в `master`. Через 1–2 минуты сайт обновится.

## Способ 2. Вручную (3 команды)

Если хотите сделать сами — открываете терминал в папке проекта и выполняете по очереди:

```bash
git add -A
git commit -m "коротко что поменяли"
git push origin HEAD:master
```

- `git add -A` — берёт все изменённые файлы.
- `git commit -m "..."` — сохраняет их одной «пачкой» с подписью.
- `git push origin HEAD:master` — отправляет в прод.

Через 1–2 минуты сайт по адресу выше обновится. Проверить, что всё ушло, можно так:

```bash
git log origin/master --oneline -1
```

Должна показаться ваша последняя правка.

---

## Если кто-то правил сайт параллельно

Иногда прод успевают обновить из другой сессии, и `push` ругается (`rejected`, `fast-forward`). Тогда сначала подтяните чужие изменения, потом снова отправьте свои:

```bash
git fetch origin
git rebase origin/master
git push origin HEAD:master
```

Если при этом появятся конфликты (git пишет `CONFLICT`) — не гадайте, просто попросите Claude: «разреши конфликты и залей».
